# 半自动 Landmark 区域对齐方案

> 创建于 2026-04-29，面向 Page3 / ModelAssemble 的模型对齐增强设计。
> 核心目标：用户在 source 和 target 上各放置一个 landmark 种子点，系统自动扩张区域、分析结构相似性、建议对应候选，用户确认后完成对齐。

---

## 一、背景与现状

### 当前对齐流程（SVD 全局对齐）

```
手工点选 3+ landmark 对  →  computeLandmarkAlignment()  →  alignSourceMeshByLandmarks()  →  全局平移/旋转/缩放
```

- 支持 rigid / similarity 两种模式
- 需要手动在 source 和 target 上精确对应地点 3 个以上的点
- 每次结果需依赖点位质量，点位偏差直接影响 RMSE

### 现有代码基础

| 模块 | 文件 | 可复用内容 |
|------|------|-----------|
| SVD 对齐 | `src/three/alignment.ts` | `computeLandmarkAlignment()`、`alignSourceMeshByLandmarks()`、`applyTransform()`、向量辅助函数 |
| Landmark 状态 | `src/three/landmarkStore.ts` | `addSrcLandmark()`、`transformSrcLandmarks()`、`isBalanced()` |
| 类型定义 | `src/three/types.ts` | `Vec3`、`Face3`、`LandmarkPoint` |
| 页面集成 | `src/pages/Page3/ModelAssemble.tsx` | 双视口、结果叠加视图、误差面板 |
| Mesh 加载 | `src/three/glbLoader.ts` | `loadGlbAsMesh()` → `{ vertices, faces }` |

### 关键缺口

| 能力 | 现状 |
|------|------|
| 网格邻接图 | ❌ 无，faces 仅存为三元组数组 |
| 连通域扩张 | ❌ 无 BFS/区域 grow |
| 结构特征描述子 | ❌ 无曲率、法线直方图等 |
| 候选自动匹配 | ❌ 无，全靠手工 |
| 单点触发区域 | ⚠️ UI 可点单点，但无后续区域处理 |

---

## 二、用户交互目标

```
用户点 source 上一个大概位置  →  系统扩张区域（蓝色高亮）
用户点 target 上一个大概位置  →  系统扩张区域（橙色高亮）
          ↓
系统分析两个区域的结构相似性，建议 3-8 个候选对应点对（带置信度）
          ↓
用户逐对确认 / 拒绝 / 手动调整
          ↓
确认的点对写入 landmark store，触发现有 SVD 对齐流程
```

设计原则：**建议，不强制**。系统给出候选，最终决策永远在用户手中；低置信度自动回退手工流程。

## 三、分阶段开发计划

### Phase 1：网格连通性基础

**目标**：加载 mesh 后立即构建邻接结构，缓存到页面状态，供后续所有步骤使用。

**邻接图构建**（TypeScript，O(faces×3)，10万面约 1ms）：

```ts
// 顶点→邻居顶点
function buildVertexAdjacency(faces: Face3[]): Map<number, Set<number>> {
  const adj = new Map<number, Set<number>>();
  for (const [a, b, c] of faces) {
    if (!adj.has(a)) adj.set(a, new Set()); adj.get(a)!.add(b); adj.get(a)!.add(c);
    if (!adj.has(b)) adj.set(b, new Set()); adj.get(b)!.add(a); adj.get(b)!.add(c);
    if (!adj.has(c)) adj.set(c, new Set()); adj.get(c)!.add(a); adj.get(c)!.add(b);
  }
  return adj;
}

// 顶点→所在面（用于法线、面积计算）
function buildVertexFaces(faces: Face3[]): Map<number, number[]> {
  const vf = new Map<number, number[]>();
  for (let i = 0; i < faces.length; i++) {
    for (const v of faces[i]) {
      if (!vf.has(v)) vf.set(v, []);
      vf.get(v)!.push(i);
    }
  }
  return vf;
}
```

**新增类型**（扩展 `src/three/types.ts`）：

```ts
export interface MeshAdjacency {
  vertexNeighbors: Map<number, Set<number>>;  // 顶点→邻居
  vertexFaces: Map<number, number[]>;          // 顶点→面索引
}
```

**接入点**：`ModelAssemble.tsx` 中 `loadGlbAsMesh()` 完成后立即构建，存入 `srcAdj` / `tarAdj` state。

### Phase 1：连通域扩张（Region Grow）

**目标**：用户点击 seed → BFS 扩张 → 生成区域顶点集合 → 高亮显示。

**BFS 实现**（带两种停止条件）：

```ts
function regionGrowBFS(
  seedVertex: number,
  adjacency: Map<number, Set<number>>,
  vertices: Vec3[],
  options: {
    maxSteps?: number;           // 最大 BFS 层数（默认 15）
    maxVertices?: number;        // 区域最大顶点数（默认 2000）
    curvatureThreshold?: number; // 法线偏转角阈值（弧度，默认 π/3）
    vertexNormals?: Vec3[];      // 若提供，用于曲率剪枝
  } = {}
): Set<number> {
  const { maxSteps = 15, maxVertices = 2000, curvatureThreshold = Math.PI / 3, vertexNormals } = options;
  const visited = new Set<number>([seedVertex]);
  let frontier = [seedVertex];
  const seedNormal = vertexNormals?.[seedVertex];

  for (let step = 0; step < maxSteps && frontier.length > 0; step++) {
    const nextFrontier: number[] = [];
    for (const v of frontier) {
      for (const neighbor of adjacency.get(v) ?? []) {
        if (visited.has(neighbor)) continue;
        if (seedNormal && vertexNormals) {
          const n = vertexNormals[neighbor];
          const cos = seedNormal[0]*n[0] + seedNormal[1]*n[1] + seedNormal[2]*n[2];
          if (cos < Math.cos(curvatureThreshold)) continue;
        }
        visited.add(neighbor);
        nextFrontier.push(neighbor);
        if (visited.size >= maxVertices) return visited;
      }
    }
    frontier = nextFrontier;
  }
  return visited;
}
```

**停止条件**（双条件优于固定顶点数，泛化更好）：
- 条件 A：BFS 层数达到 `maxSteps`
- 条件 B：顶点总数达到 `maxVertices`
- 条件 C（可选）：遇到曲率突变（法线夹角超阈值）自动停止扩张

**新增类型**：

```ts
export interface MeshRegion {
  seedVertex: number;
  vertices: Set<number>;       // 区域顶点集
  centroid: Vec3;              // 几何中心
  boundingRadius: number;      // 包围球半径（归一化用）
}
```

### Phase 2：区域结构描述子（Lite）

**目标**：对每个区域生成轻量特征向量，用于候选打分，不引入重型算法。

**描述子组成（维度 ~32）**：

| 分量 | 维度 | 计算方式 | 作用 |
|------|------|---------|------|
| 法线直方图 | 12 | 将法线方向按球面分 12 桶，统计频率 | 区域整体朝向分布 |
| 测地距离直方图 | 8 | 从 seed 出发的 BFS 层统计面积 | 区域形状轮廓 |
| 面积占比 | 1 | region 总面积 / 全局 bbox 面积 | 区域相对规模 |
| seed 邻域曲率 | 4 | 顶点 1/2/3/4 圈法线平均偏转角 | 种子点局部形状 |
| 主轴 PCA | 3+4 | region 顶点做 PCA，取前两轴方向+比例 | 区域拉伸方向 |

**注意**：GLB 常见大量重复顶点（缝合处），在描述子计算前可做轻量 welding 视图（合并距离 < ε 的顶点），仅用于分析，**不改变原始网格索引**。

---

### Phase 2：候选匹配与置信度

**目标**：给定 source region 的 seed，在 target region 内找 top-k 候选对应点，并给出可解释的置信度。

**核心思路**（借鉴 NRSC 仓库的 Karcher mean + FPS 思想，在 TS 中轻量实现）：

1. 在 source region 中用 farthest-in-region 采样（基于 BFS 层级距离），取 3-5 个代表点
2. 对每个代表点计算局部描述子
3. 在 target region 中搜索描述子最近邻，输出 top-k 候选
4. 按描述子距离 + 几何距离联合打分，输出置信度（0~1）

**置信度计算**：

```
confidence = 1 - normalize(descriptor_distance + λ * geometric_distance)
```

- 置信度 < 0.5：UI 标为"低可信，建议手工确认"
- 置信度 < 0.3：自动降级为纯手工 landmark 流程

**输出结构**：

```ts
export interface LandmarkCandidate {
  srcVertex: number;
  tarVertex: number;
  confidence: number;           // 0~1
  descriptorDist: number;       // 原始描述子距离（可解释）
  geometricDist: number;        // 欧式距离（归一化）
  suggestAccept: boolean;       // confidence >= 0.5 时自动建议接受
}
```

### Phase 3：对齐集成

**目标**：从建议点对中组装 3-8 对 landmark，复用现有 SVD 流程，完成对齐。

- 入口：用户确认候选后，`addSrcLandmark()` / `addTarLandmark()` 写入 store（与手工流程完全一致）
- 不修改现有 `computeLandmarkAlignment()` 和 `alignSourceMeshByLandmarks()` 接口
- 允许用户在自动建议的基础上继续手工补点，混合使用
- 误差面板（RMSE/mean/max）继续正常工作

### Phase 3：可视化与控制

**新增 UI 元素**：

| 元素 | 描述 |
|------|------|
| 蓝色区域高亮 | source region grow 结果（半透明叠加） |
| 橙色区域高亮 | target region grow 结果 |
| 候选点标记 | 带序号和置信度的点标，高置信 = 绿色，低置信 = 黄色 |
| 确认/拒绝按钮组 | 逐对确认，被确认点转为普通 landmark 颜色 |
| 重算按钮 | 重新从 seed 扩张或重新匹配，不清除已确认 landmark |
| 回退按钮 | 清空候选，回到纯手工 landmark 模式 |

---

### Phase 4：边界条件处理与验证

**Guardrails**：

- 区域过小（< 10 顶点）：提示"区域太小，请往更大的结构上点击"
- 区域过大（> 5000 顶点）：截断并提示"区域已截断，建议在更局部的位置点击"
- 非流形边（一条边被 3 个以上面共享）：BFS 跳过非流形邻居，避免扩张到意外区域
- 全为重复顶点（seam）：welding 后 region 为空，提示"该位置顶点分层，请移动到实体面上"
- 置信度全低：自动降级手工流程，状态栏提示原因

**验收标准**：

1. 邻接构建和 BFS 在 Demo 网格（10 顶点，16 面）结果稳定
2. source/target 各点 1 个 seed → 可看到扩张区域与 top-k 候选（< 500ms）
3. 确认候选后对齐，RMSE/mean/max 在误差面板正常更新
4. 人物模型左右手对称结构对齐成功率 > 80%（测试 3 个不同姿态组合）
5. 低置信度场景自动降级，不影响原手工流程

## 四、外部参考：ZHLRJ/Non-Rigid-Shape-Correspondence

**仓库链接**：https://github.com/ZHLRJ/Non-Rigid-Shape-Correspondence

### 该仓库做了什么

| 模块 | 技术 | 用途 |
|------|------|------|
| `geometricMedian/` | Karcher mean + VectorHeatMethod | 在曲面上找"内蕴重心"，用作稳定 landmark 候选 |
| `logarithmicMap/` | 内蕴对数映射（Log map） | 把曲面局部展开到切空间，用于可视化和邻域分析 |
| `pyFM/` | Functional Map + HKS/WKS 描述子 | 稠密点对点对应关系计算 |
| `tracinggeodesic/` | MMP 测地线算法 | 精确测地距离和路径追踪 |

### 对本方案的价值评估

| 能力 | 价值 | 说明 |
|------|------|------|
| Karcher mean 思路 | ⭐⭐⭐ 高 | 可借鉴到"区域中心稳定化"：用加权测地中心代替几何平均，在非均匀网格上更稳定 |
| Geodesic FPS | ⭐⭐⭐ 高 | 区域内候选采样策略：按测地距离做 farthest point sampling，保证覆盖均匀 |
| HKS / WKS 描述子 | ⭐⭐ 中 | Phase 2 描述子的升级方向，需要 Laplace-Beltrami 特征值，计算量偏高 |
| Functional Map | ⭐ 低（当前阶段） | 面向 dense correspondence，适合"高自动"版本，当前半自动阶段暂不需要 |
| `geometry-central` / `pyFM` 依赖 | ❌ 不引入 | C++/Python 运行时，与前端 TS/React 链路不兼容 |

### 借鉴策略

**第一版只迁移思想，不搬实现**：

1. **Geodesic FPS 轻量版**：用 BFS 层级距离近似测地距离，在区域内做分层 farthest-point 采样，选出 3-5 个代表候选点
2. **区域中心稳定化**：用区域内顶点面积加权平均（而非简单欧式均值）作为区域中心，模拟 Karcher mean 多数情况下的行为
3. **HKS/WKS 升级路径**：Phase 2 先用法线 + 测地频率直方图，Phase 4 视效果决定是否升级为 Laplacian 特征谱描述子
